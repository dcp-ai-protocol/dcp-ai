Why DCP?

Trust infrastructure for autonomous agents

DCP provides the cryptographic identity, policy gating, and audit chain that every AI agent needs to operate responsibly in the real world.

🔐

Post-Quantum Security

Hybrid Ed25519 + ML-DSA-65 composite signatures with cryptographic binding. PQ signatures cover classical signatures, preventing stripping attacks. Ready for the quantum era today.

🔗

P2P Verification

No central authority required. Any party can verify a DCP bundle independently using the embedded keys and composite signatures. Decentralized trust by design.

📑

Full Audit Trail

Dual-hash chains (SHA-256 + SHA3-256) with PQ checkpoints create an immutable, tamper-evident record of every AI action. Merkle roots seal each bundle.

1. Bind Identity

A Responsible Principal Record (RPR) ties a real person to an Agent Passport using dual Ed25519 + ML-DSA-65 keypairs. This creates a non-repudiable chain of responsibility.

2. Declare Intent

Before any sensitive action, the agent declares intent: what it wants to do, which data classes are involved, and the estimated impact. A policy engine gates the action.

3. Audit Everything

Every action produces an audit entry with dual-hash chains. PQ checkpoints periodically seal the chain with full hybrid signatures for long-term integrity.

4. Seal the Bundle

All artifacts are assembled into a Citizenship Bundle with a cryptographic manifest. The bundle is composite-signed, creating a portable proof of authorized AI activity.

Quick Start

Up and running in 60 seconds

Initialize a DCP identity and create your first signed bundle with just a few commands.

Terminal
# Install and initialize
npx @dcp-ai/cli init

# Creates: .dcp/config.json, .dcp/keys/, .dcp/identity.json, .dcp/passport.json
TypeScript
import {
  generateKeypair, signObject, verifyObject,
  BundleBuilder, signBundle
} from '@dcp-ai/sdk';

// Generate an Ed25519 keypair
const keys = generateKeypair();

// Build a citizenship bundle
const bundle = new BundleBuilder()
  .responsiblePrincipalRecord(rpr)
  .agentPassport(passport)
  .intent(intent)
  .policyDecision(policy)
  .addAuditEntry(audit)
  .build();

// Sign and verify
const signed = signBundle(bundle, keys.secretKeyB64);
const result = verifySignedBundle(signed);
console.log(result.verified); // true

Adaptive Security

Four tiers, zero configuration

DCP automatically selects the appropriate cryptographic tier based on risk score, data classification, and action type.

Tier 0 — Routine

Routine

  • Ed25519 classical signatures only
  • PQ checkpoint every 50 events
  • Low-risk browsing and reads
  • Score < 200
Tier 1 — Standard

Standard

  • Ed25519 + hybrid preferred
  • PQ checkpoint every 10 events
  • API calls and general writes
  • Score 200–499
Tier 2 — Elevated

Elevated

  • Full hybrid per-operation required
  • PQ checkpoint every event
  • PII, financial data, payments
  • Score 500–799
Tier 3 — Maximum

Maximum

  • Full hybrid + immediate verification
  • PQ checkpoint every event
  • Credentials, children's data
  • Score ≥ 800
